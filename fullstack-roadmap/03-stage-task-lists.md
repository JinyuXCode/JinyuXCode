# 阶段任务清单

## 阶段 1：Go 项目基础

适用周数：第 1 周。

学习任务：

- [ ] 理解 `go mod init`、`go mod tidy`、`go get`。
- [ ] 能按 package 拆分代码。
- [ ] 能设计简单 struct。
- [ ] 能写 method。
- [ ] 能用 interface 抽象依赖。
- [ ] 能返回和包装 error。
- [ ] 能读写 JSON 文件。
- [ ] 能写基础单元测试。

实践任务：

- [ ] 完成命令行 Todo。
- [ ] 给核心逻辑写测试。
- [ ] 写一次 AI code review。

验收问题：

- [ ] 这个项目为什么需要拆 package？
- [ ] 哪些错误应该向上返回？
- [ ] 哪些地方不应该用 interface？

## 阶段 2：HTTP API

适用周数：第 2 周。

学习任务：

- [ ] 理解 HTTP method。
- [ ] 理解 status code。
- [ ] 能解析 query 参数。
- [ ] 能解析 path 参数。
- [ ] 能解析 JSON body。
- [ ] 能返回 JSON。
- [ ] 能处理 CORS。
- [ ] 能写 middleware。

实践任务：

- [ ] 完成内存版用户 API。
- [ ] 写 API 文档。
- [ ] 用 Postman / Apifox / curl 测试接口。

验收问题：

- [ ] 参数错误返回 400 还是 422？
- [ ] 未登录返回 401 还是 403？
- [ ] 创建资源成功返回 200 还是 201？

## 阶段 3：Gin 与分层

适用周数：第 3 周。

学习任务：

- [ ] 理解 Gin 路由组。
- [ ] 理解 middleware 链路。
- [ ] 理解 handler / service / repository 分工。
- [ ] 理解 DTO 和数据库 model 区别。
- [ ] 能设计统一响应。
- [ ] 能设计统一错误码。

实践任务：

- [ ] 把用户 API 迁移到 Gin。
- [ ] 拆出 service。
- [ ] 拆出 repository 接口。
- [ ] 加 request id。

验收问题：

- [ ] handler 中是否出现业务规则？
- [ ] service 中是否出现 HTTP response？
- [ ] repository 是否泄漏 ORM 细节？

## 阶段 4：数据库与 SQL

适用周数：第 4 到第 5 周。

学习任务：

- [ ] 理解主键。
- [ ] 理解唯一索引。
- [ ] 理解普通索引。
- [ ] 理解一对多。
- [ ] 理解多对多。
- [ ] 理解事务。
- [ ] 理解分页查询。
- [ ] 理解数据库迁移。

实践任务：

- [ ] 用户中心接入数据库。
- [ ] 写迁移文件。
- [ ] 写分页查询。
- [ ] 处理唯一冲突。
- [ ] 写 repository 测试。

验收问题：

- [ ] 哪些约束必须由数据库保证？
- [ ] 哪些查询需要索引？
- [ ] 哪些操作需要事务？

## 阶段 5：认证鉴权

适用周数：第 6 到第 7 周。

学习任务：

- [ ] 理解密码哈希。
- [ ] 理解 JWT 或 Session。
- [ ] 理解认证和鉴权区别。
- [ ] 理解 RBAC。
- [ ] 理解接口权限和菜单权限区别。

实践任务：

- [ ] 实现注册。
- [ ] 实现登录。
- [ ] 实现认证 middleware。
- [ ] 实现角色管理。
- [ ] 实现权限校验。

验收问题：

- [ ] 密码是否明文存储？
- [ ] token 里是否放了敏感信息？
- [ ] 后端是否真正校验权限？

学习任务：

- [ ] 能从业务需求反推表结构。
- [ ] 能设计后台接口和前台接口。
- [ ] 能处理分页、筛选、搜索。
- [ ] 能处理文件上传。

实践任务：

- [ ] 完成 CMS 管理端。
- [ ] 完成 CMS 前台。
- [ ] 完成图片上传。
- [ ] 完成接口文档。

验收问题：

- [ ] 前台是否能访问草稿？
- [ ] 后台接口是否需要登录？
- [ ] 上传文件是否有限制？

- [ ] 理解异步任务。
- [ ] 理解操作日志。

实践任务：

- [ ] 文章详情缓存。
- [ ] 热门文章排行。
- [ ] 登录失败限制。
