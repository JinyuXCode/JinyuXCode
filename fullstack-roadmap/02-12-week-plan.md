# 12 周执行计划

## 使用方式

每周按 4 个动作推进：

1. 学核心概念。
2. 做一个可运行功能。
3. 让 AI review。
4. 写复盘。

不要先看完整知识大全。每周只学完成项目所需的最小知识，然后在项目中补齐。

## 第 1 周：Go 后端项目基本手感

目标：把 Go 从基础语法推进到能组织小项目。

任务：

- [ ] 创建 Go module。
- [ ] 学会 package 拆分。
- [ ] 写 struct、method、interface。
- [ ] 练习 error 返回和包装。
- [ ] 读写 JSON 文件。
- [ ] 写 3 到 5 个单元测试。

交付物：

- 命令行 Todo v1。
- 需求拆解文档。
- 单元测试结果。

验收：

- 能解释每个 package 的职责。
- 能说明错误在哪里处理。
- 能独立修改一个字段或命令。

## 第 2 周：HTTP 与 RESTful API

目标：理解后端服务入口。

任务：

- [ ] 使用 `net/http` 写接口。
- [ ] 处理 query、path、JSON body。
- [ ] 设计统一响应结构。
- [ ] 处理常见状态码。
- [ ] 加 CORS middleware。
- [ ] 加日志 middleware。

交付物：

- 内存版用户管理 API。
- API 文档。
- 接口测试记录。

验收：

- 前端能调用接口。
- 错误响应统一。
- 能解释 400、401、403、404、409、500 的区别。

## 第 3 周：Gin 与项目分层

目标：从能写接口升级到能组织项目。

任务：

- [ ] 使用 Gin 路由组。
- [ ] 拆 handler / service / repository。
- [ ] 设计 DTO 和 model。
- [ ] 加统一错误码。
- [ ] 加 request id 日志。
- [ ] 写简单配置加载。

交付物：

- 用户中心 API v1。
- 项目目录结构说明。
- 错误码表。

验收：

- handler 不写业务规则。
- service 不依赖 HTTP 细节。
- repository 可替换成数据库实现。

## 第 4 周：数据库建模与 SQL

目标：能判断 AI 生成的表设计是否靠谱。

任务：

- [ ] 设计 `users` 表。
- [ ] 学主键、唯一索引、普通索引。
- [ ] 写基础 CRUD SQL。
- [ ] 支持分页、搜索、排序。
- [ ] 使用迁移工具管理 schema。
- [ ] 处理唯一约束冲突。

交付物：

- 用户中心数据库版。
- 数据库迁移文件。
- 表设计说明。

验收：

- 邮箱或手机号唯一由数据库保证。
- 列表查询不会无意识全表扫描。
- 能解释为什么某些字段需要索引。

## 第 5 周：GORM / sqlc 与 Repository

目标：掌握 Go 操作数据库的工程写法。

任务：

- [ ] 在 GORM 和 sqlc 中选一个。
- [ ] 配置数据库连接池。
- [ ] repository 方法接收 `context.Context`。
- [ ] 数据库错误转换成业务错误。
- [ ] 给 repository 或 service 写测试。

交付物：

- 用户中心持久化重构。
- repository 接口说明。
- 基础测试。

验收：

- handler 不直接操作数据库。
- 数据库配置不硬编码。
- 错误不会把数据库细节暴露给前端。

## 第 6 周：认证、登录态与安全基础

目标：做出真实项目必须有的登录系统。

任务：

- [ ] 注册时密码哈希。
- [ ] 登录返回 JWT 或设置 Cookie Session。
- [ ] 实现认证 middleware。
- [ ] 实现 `GET /me`。
- [ ] 保护需要登录的接口。
- [ ] 配置 secret 和过期时间。

交付物：

- 用户中心登录注册。
- 登录态设计说明。
- 安全检查记录。

验收：

- 数据库没有明文密码。
- JWT secret 不在代码里硬编码。
- 未登录不能访问受保护接口。

## 第 7 周：RBAC 权限与后台管理

目标：理解权限边界。

任务：

- [ ] 设计用户、角色、权限表。
- [ ] 支持给用户分配角色。
- [ ] 支持给角色分配权限。
- [ ] middleware 校验接口权限。
- [ ] 设计前端菜单权限数据。

交付物：

- 权限后台 v1。
- 权限矩阵。
- 越权测试记录。

验收：

- 权限不只在前端控制。
- 普通用户不能访问管理员接口。
- 能解释菜单权限和接口权限的区别。

## 第 8 周：CMS 内容管理系统

目标：完成第一个完整前后端项目。

任务：

- [ ] 设计文章、分类、标签表。
- [ ] 实现文章草稿、发布、下架。
- [ ] 实现管理端文章 CRUD。
- [ ] 实现前台文章列表和详情。
- [ ] 实现图片上传。
- [ ] 完成前后端联调。

交付物：

- CMS v1。
- API 文档。
- 数据库 ERD。

验收：

- 草稿不会出现在前台。
- 列表支持分页、搜索、筛选。
- 管理端接口需要登录。

## 第 9 周：Redis、缓存与限流

目标：知道 Redis 什么时候该用。

任务：

- [ ] 缓存文章详情。
- [ ] 设计缓存 key。
- [ ] 文章更新后删除缓存。
- [ ] 实现热门文章排行。
- [ ] 实现登录失败次数限制。
- [ ] 实现简单 IP 限流。

交付物：

- CMS 缓存版。
- Redis key 设计文档。
- 缓存失效说明。

验收：

- 更新文章后不会长期读到旧数据。
- Redis 挂了不会导致核心服务完全不可用。
- 限流返回明确状态码。

## 第 10 周：文件上传、异步任务与通知

目标：补齐真实项目常见非 CRUD 能力。

任务：

- [ ] 限制上传文件类型和大小。
- [ ] 保存文件元信息。
- [ ] 实现媒体库。
- [ ] 实现操作日志。
- [ ] 模拟异步任务。
- [ ] 模拟通知或邮件发送。

交付物：

- CMS 媒体库。
- 操作日志表。
- 异步任务说明。

验收：

- 文件名安全处理。
- 删除资源时有清晰策略。
- 异步任务失败有记录。

## 第 11 周：测试、Docker 与部署

目标：从本地能跑推进到别人也能跑。

任务：

- [ ] 给核心 service 写测试。
- [ ] 写接口测试用例。
- [ ] 编写 Dockerfile。
- [ ] 编写 docker-compose。
- [ ] 提供 `.env.example`。
- [ ] 部署到服务器或云平台。

交付物：

- 可容器化启动的 CMS。
- 部署说明。
- 测试报告。

验收：

- `docker-compose up` 能启动服务、数据库、Redis。
- secret 不提交到仓库。
- 容器重启后数据不丢失。

## 第 12 周：综合项目二选一

目标：做一个更接近真实业务的作品集项目。

选项：

- 任务协作系统。
- 轻量电商后台。

任务：

- [ ] 写完整需求拆解。
- [ ] 画实体关系。
- [ ] 设计状态流转。
- [ ] 标记事务场景。
- [ ] 标记权限规则。
- [ ] 完成核心链路。
- [ ] 写复盘文档。

交付物：

- 综合项目 MVP。
- 需求拆解文档。
- 架构说明。
- 关键测试记录。

验收：

- 至少有一个复杂状态流转。
- 至少有一个事务场景。
- 至少有一个权限矩阵。
- 能说明 AI 生成代码中你审查和修改了什么。
