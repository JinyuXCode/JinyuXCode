# 项目 1：命令行 Todo

## 项目定位

这是第一个项目，不追求复杂功能。目标是把 Go 基础语法变成项目手感。

你要练的是：

- package 拆分。
- struct 设计。
- error 处理。
- JSON 持久化。
- 单元测试。

## 功能范围

- [ ] 新增任务。
- [ ] 完成任务。
- [ ] 删除任务。
- [ ] 列出所有任务。
- [ ] 按状态筛选任务。
- [ ] 使用 JSON 文件保存数据。

## 推荐数据结构

```text
Todo
- ID
- Title
- Done
- CreatedAt
- UpdatedAt
```

## 推荐目录

```text
todo-cli/
  cmd/
  internal/
    todo/
    storage/
  go.mod
  README.md
```

## 需求拆解任务

- [ ] 定义 Todo 字段。
- [ ] 定义命令列表。
- [ ] 定义 JSON 文件路径。
- [ ] 定义错误场景。
- [ ] 定义测试场景。

## 可以让 AI 做

- [ ] 生成初版目录结构。
- [ ] 生成命令行参数解析代码。
- [ ] 生成 JSON 读写代码。
- [ ] 生成单元测试样例。

## 你必须审查

- [ ] 错误是否被吞掉。
- [ ] 文件不存在时是否能初始化。
- [ ] JSON 损坏时如何提示。
- [ ] 测试是否依赖真实用户目录。
- [ ] package 是否拆得过度。

## 最小验收标准

- [ ] `add` 能新增任务。
- [ ] `done` 能完成任务。
- [ ] `list` 能展示任务。
- [ ] 程序重启后任务仍存在。
- [ ] 核心逻辑有测试。

## 进阶加分
