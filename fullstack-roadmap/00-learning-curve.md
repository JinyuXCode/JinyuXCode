# 学习曲线与能力目标

## 总原则

你已经有前端经验，刚学完 Go 基础语法，所以路线不应该从“背语法大全”开始，而应该从“能交付后端功能”开始。

学习重点按优先级排序：

1. 需求拆解。
2. 数据建模。
3. API 设计。
4. 后端分层。
5. 认证鉴权。
6. 测试验证。
7. 部署交付。
8. 性能、安全、可观测性。

语法和库函数只需要达到“能读懂、能改、能判断 AI 代码是否靠谱”的程度。常用 API 会在项目中自然重复出现，不需要脱离项目死记。

## 从前端到全栈的关注点变化

| 前端常见关注点 | 全栈 / 后端要补齐的关注点 |
| --- | --- |
| 页面状态 | 业务状态 |
| 表单校验 | 业务规则和数据约束 |
| 调接口 | 设计接口 |
| 路由权限 | 接口权限和数据权限 |
| 请求 loading | 超时、重试、幂等 |
| UI 错误提示 | 统一错误码和日志定位 |
| 本地状态管理 | 数据一致性、事务、缓存 |
| 打包部署前端 | 服务进程、数据库、Redis、Nginx、Docker |

## 连贯学习曲线

### 第 1 层：Go 项目手感

目标：让 Go 不再停留在语法题，而是能组织一个小项目。

要掌握：

- `go mod`。
- package 拆分。
- struct / interface / method。
- error 返回和包装。
- JSON 文件读写。
- 单元测试。

对应项目：命令行 Todo。

不要急着学：

- 微服务。
- 消息队列。
- 复杂 ORM。
- 云原生。

### 第 2 层：HTTP API

目标：理解后端服务如何接收请求、返回响应、处理错误。

要掌握：

- HTTP method。
- status code。
- request body。
- query / path 参数。
- JSON 响应。
- CORS。
- middleware。
- graceful shutdown。

对应项目：内存版用户管理 API。

### 第 3 层：Gin 与分层架构

目标：从“能写接口”变成“能维护项目”。

要掌握：

- Gin 路由组。
- handler / service / repository。
- DTO 和 model 区分。
- 统一错误响应。
- 日志 middleware。
- 配置管理。

对应项目：用户中心 API v1。

### 第 4 层：数据库与持久化

目标：能判断 AI 生成的数据库结构是否可靠。

要掌握：

- 表设计。
- 主键、唯一索引、普通索引。
- 一对多、多对多。
- 事务。
- 分页、排序、筛选。
- 数据库迁移。

对应项目：用户中心接入 PostgreSQL / MySQL。

### 第 5 层：认证、鉴权与安全

目标：做出真实后台系统的登录和权限。

要掌握：

- 密码哈希。
- JWT 或 Cookie Session。
- 认证 middleware。
- 当前用户上下文。
- RBAC。
- 接口级权限控制。

对应项目：用户中心 + 权限后台。

### 第 6 层：完整前后端项目

目标：做出可展示的第一个完整作品。

要掌握：

- 前后端联调。
- API 文档。
- 文章、分类、标签建模。
- 图片上传。
- 管理端权限。
- Redis 缓存。
- Docker 部署。

对应项目：CMS 内容管理系统。

### 第 7 层：复杂业务系统

目标：证明你能处理非 CRUD 的真实业务。

要掌握：

- 状态流转。
- 事务边界。
- 并发冲突。
- 权限矩阵。
- 操作日志。
- 业务审计。

对应项目：任务协作系统或电商后台。

## 学习曲线检查

如果你发现自己卡住，按下面方式判断原因：

| 卡点 | 说明 | 补救方式 |
| --- | --- | --- |
| 代码能跑但不知道为什么 | 基础模型不清楚 | 回到 Go / HTTP / SQL 核心概念 |
| 能写 CRUD 但不会设计表 | 数据建模不足 | 先画实体关系，再让 AI 生成 SQL |
| 登录能做但权限混乱 | 权限边界不清 | 先列角色、资源、动作，再写代码 |
| AI 生成代码很多但你不敢改 | 分层不清 | 强制按 handler / service / repository 拆 |
| 项目越来越乱 | 缺少验收标准 | 每个功能先写接口、测试、错误码 |

## 最终能力目标

完成这条路线后，你应该能做到：

- 独立拆解一个后台功能。
- 设计表结构和 API。
- 让 AI 生成初版代码。
- 审查 AI 代码中的安全、权限、事务和性能问题。
- 本地运行、测试、调试。
- 用 Docker 启动完整环境。
- 把项目部署到服务器或云平台。
- 用作品集说明你的设计思路，而不只是展示页面。
